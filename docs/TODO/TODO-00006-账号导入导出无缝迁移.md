## 风险与注意事项

| 风险点 | 应对措施 |
|---|---|
| auto 模式允许 `group_id=null`，可能破坏旧入口校验逻辑 | 将 group 校验逻辑下沉到 auto 分支内部，其他分支保持不变 |
| 覆盖更新误操作 | UI 明示“覆盖更新”风险 + 审计记录覆盖对象与字段名（不记录值） |
| GPTMail API 不稳定/限流 | mock 覆盖测试；导入限流 + 单次导入数量上限 |
| 导出文件包含明文敏感信息 | 保持 2FA 二次验证不变；审计/日志不输出敏感字段 |
| 旧导出文件兼容性 | auto 导入跳过 `#` 注释行；旧文件无头部也可直接导入 |
